# VLAW Metric Contract

## Canonical Fields

### `results_base.json` (heuristic baseline)
- `result_type`: `baseline_heuristic`
- `task`: string
- `success_rate`: float in `[0,1]`
- `episodes`: integer
- `provenance`: object

### `results_base_real.json` (OpenPI baseline)
- `result_type`: `baseline_openpi_real`
- `task`: string
- `success_rate`: float in `[0,1]`
- `episodes`: integer
- `provenance`: object

### `results_vlaw.json` (loop outputs)
- `result_type`: `vlaw_loop`
- `task`: string
- `metrics`: object `{metric_name: float}`
- `metrics_meta`: object `{metric_name: {backend, episodes, ...}}`
- `provenance`: object

## Semantic Policy
- `Base` in `results_vlaw.json` is heuristic baseline generated by loop policy wrapper.
- `Base-Real` is injected only when `results_base_real.json` exists and is parseable.
- Plots may include both to make backend differences explicit.
